機能

シンプルな操作で、必要なことだけをやります。

3フォーマット対応
JPG・PNG・WebP に変換して保存。元画像の形式を問わず変換できます。
右クリックだけ
画像を右クリック→フォーマット選択→保存。追加の操作は不要です。
画像データはローカル完結
画像の変換処理はすべてブラウザ内で実行され、画像データが外部へ出ることはありません。品質監視のための匿名メトリクス送信についてはプライバシーポリシーをご覧ください。
権限の最小化
host_permissions 不使用。必要最小限の4権限のみで動作し、ユーザー操作時のみアクセス権を取得します。
アニメーション対応
アニメーションGIF・WebPは変換せず元の形式のまま保存。SVGも同様です。
フォールバック保存
変換に失敗した場合でも、元画像をそのままダウンロードします。保存に失敗しません。
ファイル名を自動生成
元画像のURLからファイル名を取得。取得できない場合は日時形式で自動命名します。

使い方

3ステップで変換・保存できます。

1

画像を右クリック

Webページ上の保存したい画像を右クリックしてコンテキストメニューを開きます。

2

フォーマットを選択

「画像を変換して保存」を選び、保存したい形式（PNG / JPG / WebP）をクリックします。

3

保存先を確認して保存

「名前を付けて保存」ダイアログが表示されます。ファイル名と保存先を確認して保存してください。

変換できないケース
  • chrome:// / moz-extension:// など特殊ページ — ブラウザのセキュリティ制約によりスクリプト注入不可。元の形式のままダウンロードされます。
  • アニメーションGIF / アニメーションWebP — 変換するとアニメーションが失われるため、元の形式のまま保存します。
  • SVG画像 — 変換せず元の形式のまま保存します。
  • CORS制限がある画像 — 変換に失敗した場合は元画像URLをそのままダウンロードします（フォールバック）。

技術仕様

シンプルな構成で、拡張機能本体は外部依存ゼロのバニラJavaScriptで実装されています。

変換エンジン
OffscreenCanvas + convertToBlob() でページ内変換。 content script を isolated world で注入し、ページ内の <img> 要素から Canvas に描画します。
変換品質
JPG・WebP は品質 92% で出力。人間の目でほぼ劣化が分からず、ファイルサイズを効率的に抑えるバランスポイントです。PNG はロスレス。
必要な権限
contextMenus（メニュー登録）、downloads（ダウンロード）、 activeTab（アクティブタブへのアクセス）、scripting（content script注入）の4つのみ。 host_permissions は使用しません。
セキュリティ
URLスキームを http: / https: / data: / blob: のみに制限。 ファイル名はパストラバーサル防止・nullバイト除去・特殊文字除去のサニタイズを適用。 配布ZIPには GitHub Attestation で署名しています。
品質監視
変換結果・エラー分類・バージョン番号のみを Google Analytics 4 へ匿名送信しています。URL・ファイル名・画像データは含まれません。詳細はプライバシーポリシーをご覧ください。
アーキテクチャ
メイン処理は Service Worker（background.js）に集約したシンプル構成です（拡張機能のポップアップや docs/ 配下のデモUIなどの画面は別HTMLとして実装）。 拡張機能本体は外部ライブラリ不使用。Manifest V3 準拠。
ソースコード
GitHub で公開しています。 GNU GPL v3.0 のもとで自由に使用・変更・再配布できます。派生物を配布する場合は同じく GPL v3.0 でのソースコード公開が必要です。
品質保証
GitHub Actions による CI を導入しています。main への push およびすべての PR のたびに ESLint による静的解析、ユニットテスト（Node.js 単体）、manifest.json の自動検証が実行されます。また CodeQL による静的セキュリティ解析も main への push・main 向け PR および週次スケジュールで実行されます。

よくある質問

使用上の疑問をまとめました。

chrome:// や moz-extension:// など拡張機能ページで変換できないのはなぜですか？
chrome://、chrome-extension://、moz-extension://、edge:// などの特殊なページでは、ブラウザのセキュリティ制約によりスクリプトを注入できません。そのため変換は行われず、元の形式のままダウンロードされます。通常のWebページ（https://）では正常に変換できます。
アニメーションGIFを変換しようとしたら、そのまま保存されました
アニメーションGIF・アニメーションWebPは意図的に変換しない設計です。変換するとアニメーションが失われてしまうため、元の形式のまま保存します。静止画のGIFは通常通り変換されます。
画像データは外部に送信されますか？
画像データは送信されません。変換処理はすべてブラウザ内（OffscreenCanvas）で完結しており、画像データが外部に出ることはありません。
なお、変換処理の品質監視を目的として、変換結果・エラー分類・バージョン番号のみを Google Analytics 4 へ匿名で送信しています（URL・ファイル名・画像データは含まれません）。
変換に失敗した場合はどうなりますか？
CORS制限・Canvas変換エラーなど、あらゆる失敗ケースでフォールバック処理が動作します。変換できなかった場合は元画像URLをそのままダウンロードするため、画像を取得できないケースは最小限に抑えられています。
商用利用はできますか？
GNU GPL v3.0 のもとで配布されているため、商用・非商用を問わず使用・変更・再配布できます。ただし、派生物を配布する場合は同じく GPL v3.0 でのソースコード公開が必要です。詳細はリポジトリの LICENSE を参照してください。
Firefoxには対応していますか？
Firefox 128 以上に対応しています。Firefox Add-ons（AMO）への掲載準備中のため、現在は GitHub Releases からの手動インストールをご利用ください。
